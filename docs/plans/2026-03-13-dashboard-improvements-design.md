## 기능 2 — 주간 요약 텍스트 자동 생성

### 목적
수업 종료 후 버튼 하나로 카카오톡/보고용 요약 텍스트를 클립보드에 복사.

### 구현 범위
- **변경 파일**: `dashboard.html` (프론트엔드만)
- **백엔드 변경**: 없음

### UI
- 헤더 우측에 "📋 요약 복사" 버튼 추가
- 클릭 시 아래 형식의 텍스트 생성 후 클립보드 복사 → "복사됨" 토스트

### 출력 형식
```
[기간명] 확인학습 요약
출석 N명 / 결석 N명 / 지각 N명
완료 N명 | 평균 N% | 완료율 N%
───
70% 미만: 이름1, 이름2, ...
미완료: 이름3, 이름4, ...
```
- 70% 미만, 미완료 항목은 해당자 없으면 생략

## 기능 4 — 기간별 성적 히스토리

### 목적
학생 카드 펼쳤을 때 현재 기간 + 이전 기간들의 평균 점수를 시각적으로 표시. 이번 주부터 자연 누적.

### 구현 범위
- **변경 파일**: `Code.gs` + `dashboard.html`

### 백엔드 (Code.gs)
`getDashboardData`에서 Summary 시트 스캔 시, 활성 기간 외 종료된 기간들도 함께 집계:
- `periodHistoryMap`: `{studentName: [{period, avgPct, doneCount, totalCount}]}`
- 각 학생 객체에 `history` 배열 추가 (최신순 정렬, 최대 8개)
- Summary 시트 스캔은 이미 한 번 하므로 추가 시트 읽기 없음

### 프론트엔드 (dashboard.html)
- `renderDetail(s)` 하단에 히스토리 섹션 추가
- 2주 이상 데이터 있을 때만 표시
- 표시 형식: 점수 칸 나열 (기간명 + % + 색상)

```
과거 기록
3/6회차  82%  3/13회차  91%
```

## 기능 5 — 과제별 완료율 통계

### 목적
어떤 구글폼에서 학생들이 얼마나 제출했는지 한눈에 파악.

### 구현 범위
- **변경 파일**: `dashboard.html` (프론트엔드만)
- **백엔드 변경**: 없음

### 데이터 소스
`dashboardData.students[].tasks[]` — 이미 로드된 데이터에서 계산.

### UI
- 통계 바 아래 접이식 패널 "📊 과제별 통계 ▾"
- 클릭하면 펼쳐지는 테이블:

| 과제명 | 제출 | 미제출 | 완료율 | 평균점수 |
|--------|------|--------|--------|----------|
| 유성고 1과 | 12 | 3 | 80% | 84% |

- 완료율 낮은 순 정렬 (문제 있는 과제가 위로)
- 결석 학생은 분모에서 제외
